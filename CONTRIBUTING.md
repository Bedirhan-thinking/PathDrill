# **Contributing to PathDrill**

First of all, thank you for taking the time to contribute\! 🎉

PathDrill is an open-source tool, and it grows best with community input. Whether you're fixing a typo, improving the scanning algorithm, or suggesting a completely new feature, your help is genuinely appreciated.

## **How Can I Contribute?**

### **1\. Reporting Bugs**

If you find a bug, please open an **Issue** and include:

* What you were trying to do.  
* What happened (the bug).  
* What you expected to happen.  
* Your OS and Python version.

### **2\. Suggesting Enhancements**

If you have an idea to make PathDrill faster or more useful, open an **Issue** with the tag enhancement. Describe how the feature would work and why it would be useful.

### **3\. Contributing Code (Pull Requests)**

If you want to dive into the code yourself:

1. **Fork** the repository.  
2. Create a new branch for your feature (git checkout \-b feature/amazing-feature).  
3. Make your changes. (Try to keep your code clean and add comments where necessary).  
4. Commit your changes (git commit \-m 'Add some amazing feature').  
5. Push to the branch (git push origin feature/amazing-feature).  
6. Open a **Pull Request (PR)** here.

## **Philosophy**

Keep it simple. PathDrill is designed to be lightweight and fast. When adding new features, let's try not to overcomplicate the architecture or add heavy dependencies unless absolutely necessary.

Once again, thank you for being part of this\!
